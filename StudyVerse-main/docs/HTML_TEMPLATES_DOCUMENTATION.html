1. Template Inheritance

Concept: Templates can extend other templates to avoid code duplication

Base Template (layout.html):

<!DOCTYPE html>
<html lang="en">
<head>
    <title>{% block title %}StudyVerse{% endblock %}</title>
    <link rel="stylesheet" href="style.css">
</head>
<body>
    <aside class="sidebar">
        <!-- Navigation menu -->
    </aside>
    
    <main>
        {% block content %}
        <!-- Child template content goes here -->
        {% endblock %}
    </main>
    
    {% block scripts %}
    <!-- Child template scripts go here -->
    {% endblock %}
</body>
</html>

Child Template (dashboard.html):

{% extends "layout.html" %}

{% block title %}Dashboard - StudyVerse{% endblock %}

{% block content %}
    <h1>Welcome to Dashboard</h1>
    <p>Your XP: {{ current_user.total_xp }}</p>
{% endblock %}

{% block scripts %}
    <script src="dashboard.js"></script>
{% endblock %}

Result: Child template inherits entire structure from parent!

4. Loops (For)

<!-- Loop through list -->
{% for todo in todos %}
    <div class="todo-item">
        <h3>{{ todo.title }}</h3>
        <p>Category: {{ todo.category }}</p>
        <p>Status: {{ 'Done' if todo.completed else 'Pending' }}</p>
    </div>
{% endfor %}

<!-- Empty list handling -->
{% for message in messages %}
    <div class="message">{{ message.content }}</div>
{% else %}
    <p>No messages yet.</p>
{% endfor %}

<!-- Loop with counter -->
{% for user in top_users %}
    <div class="rank-{{ loop.index }}">
        <span>#{{ loop.index }}</span>
        <span>{{ user.name }}</span>
        <span>{{ user.total_xp }} XP</span>
    </div>
{% endfor %}

<!-- Loop variables -->
{% for item in items %}
    First: {{ loop.first }}
    Last: {{ loop.last }}
    Index: {{ loop.index }}  <!-- 1-based -->
    Index0: {{ loop.index0 }} <!-- 0-based -->
    Length: {{ loop.length }}
{% endfor %}

5. Filters

Concept: Transform variables using pipe operator

<!-- String filters -->
<p>{{ username|upper }}</p>           <!-- JOHN DOE -->
<p>{{ username|lower }}</p>           <!-- john doe -->
<p>{{ username|title }}</p>           <!-- John Doe -->
<p>{{ text|truncate(50) }}</p>        <!-- First 50 chars -->

<!-- Number filters -->
<p>{{ price|round(2) }}</p>           <!-- 19.99 -->
<p>{{ count|abs }}</p>                <!-- Absolute value -->

<!-- List filters -->
<p>{{ users|length }}</p>             <!-- Count items -->
<p>{{ names|join(', ') }}</p>        <!-- Join with comma -->
<p>{{ numbers|sum }}</p>              <!-- Sum all numbers -->
<p>{{ items|first }}</p>              <!-- First item -->
<p>{{ items|last }}</p>               <!-- Last item -->

<!-- Date filters -->
<p>{{ date|date('Y-m-d') }}</p>       <!-- 2024-01-23 -->

<!-- Custom filter (from our app) -->
<p>{{ message.created_at|ist_time }}</p>

Q1: What is Jinja2 and why use it?

Answer:

Jinja2 is a modern templating engine for Python that allows us to generate dynamic HTML.

Without Jinja2 (Static HTML):

<h1>Welcome, Guest</h1>
<p>You have 0 XP</p>

With Jinja2 (Dynamic):

<h1>Welcome, {{ current_user.first_name }}</h1>
<p>You have {{ current_user.total_xp }} XP</p>

Benefits:

  • ‚úÖ Dynamic content from database
  • ‚úÖ Reusable templates (DRY principle)
  • ‚úÖ Logic in templates (if/for)
  • ‚úÖ Template inheritance

Q2: How does template inheritance work?

Answer: Parent-Child Relationship

  1. layout.html defines structure with {% block %}
  2. dashboard.html extends layout with {% extends %}
  3. Child fills in parent's blocks
  4. Result: Complete page with inherited structure

Benefits:

  • Don't repeat sidebar/navbar on every page
  • Change layout once, updates all pages
  • Consistent design across application

Q3: What are flash messages?

Answer: One-time notifications to users

In Python (app.py):

from flask import flash

@app.route('/login', methods=['POST'])
def login():
    if password_correct:
        flash('Login successful!', 'success')
    else:
        flash('Invalid password', 'danger')

In Template:

{% with messages = get_flashed_messages(with_categories=true) %}
    {% if messages %}
        {% for category, message in messages %}
            <div class="alert alert-{{ category }}">
                {{ message }}
            </div>
        {% endfor %}
    {% endif %}
{% endwith %}

Flash messages appear once, then disappear on next request!

Q4: How do forms work with Flask?

HTML Form:

<form action="{{ url_for('create_todo') }}" method="POST">
    <input type="text" name="title" required>
    <select name="category">
        <option value="study">Study</option>
        <option value="work">Work</option>
    </select>
    <button type="submit">Create Task</button>
</form>

Python Handler:

@app.route('/todos/create', methods=['POST'])
def create_todo():
    title = request.form.get('title')
    category = request.form.get('category')
    
    new_todo = Todo(title=title, category=category)
    db.session.add(new_todo)
    db.session.commit()
    
    flash('Task created!', 'success')
    return redirect(url_for('todos_page'))

Flow: User fills form ‚Üí Submit ‚Üí Python receives data ‚Üí Save to DB ‚Üí Redirect

Q5: What is CSRF protection?

Answer: Cross-Site Request Forgery Protection

Problem: Malicious websites could submit forms on your behalf!

Solution: CSRF Token

<form method="POST">
    <!-- Hidden token field -->
    <input type="hidden" 
           name="csrf_token" 
           value="{{ csrf_token() }}">
    
    <input type="text" name="username">
    <button type="submit">Submit</button>
</form>

Flask verifies token before processing form - prevents unauthorized requests!
