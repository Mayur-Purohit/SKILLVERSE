def calculate_level(total_xp):
    """
    Calculate user level based on total XP
    
    Args:
        total_xp (int): Total experience points
    
    Returns:
        int: Current level
    """
    level = 1
    xp_for_next = 100
    
    while total_xp >= xp_for_next:
        level += 1
        xp_for_next += level * 50
    
    return level

User Login:

@app.route('/login', methods=['POST'])
def login():
    email = request.form.get('email')
    password = request.form.get('password')
    
    # Find user in database
    user = User.query.filter_by(email=email).first()
    
    # Check if user exists and password is correct
    from werkzeug.security import check_password_hash
    
    if user and check_password_hash(user.password_hash, password):
        # Password correct - log user in
        login_user(user, remember=True)
        flash('Login successful!', 'success')
        return redirect(url_for('dashboard'))
    else:
        # Invalid credentials
        flash('Invalid email or password', 'danger')
        return redirect(url_for('auth'))

Security Concepts:

  • Password Hashing: Convert password to irreversible hash
  • Never store plain text passwords!
  • Session Management: Remembers logged-in users

Q2: How does password security work?

Answer:

Password Hashing Process:

  1. Registration: User enters password "mypassword123"
  2. Hashing: Algorithm converts it to: "pbkdf2:sha256:..."
  3. Storage: Only hash is stored in database
  4. Login: User enters password again
  5. Verification: Hash the input and compare with stored hash
  6. Match: If hashes match, password is correct

Why it's secure:

  • Even if database is hacked, passwords are not readable
  • Hashing is one-way (can't reverse to original)
  • Each hash is unique even for same password (salt)

Q6: How do you handle multiple users accessing database simultaneously?

Answer: Database Transactions & Locking

Problem:

Two users try to update same data at same time - could cause data corruption

Solution:

# SQLAlchemy handles this with transactions
try:
    # Start transaction
    user = User.query.get(user_id)
    user.total_xp += 50
    
    # Commit transaction (atomic operation)
    db.session.commit()
    
except Exception as e:
    # If error, rollback (undo changes)
    db.session.rollback()
    print(f"Error: {e}")
  • Transactions: All-or-nothing operations
  • ACID Properties: Atomicity, Consistency, Isolation, Durability
  • Row Locking: Database locks row during update
